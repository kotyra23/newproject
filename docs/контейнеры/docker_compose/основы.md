# 🐳 Docker Compose: Основы

## 📖 Что такое Docker Compose?

**Docker Compose** — инструмент для определения и запуска **мультиконтейнерных** приложений Docker.  
Позволяет описать всю инфраструктуру приложения (веб-сервер, БД, кэш) в **одном файле** `docker-compose.yml`.

### Ключевые преимущества:
- ✅ **Один файл конфигурации** — вся инфраструктура как код (IaC)
- ✅ **Быстрый запуск** — `docker-compose up` поднимает всё приложение
- ✅ **Изоляция сред** — dev/staging/prod через разные файлы или переменные
- ✅ **Зависимости сервисов** — автоматический порядок запуска
- ✅ **Сети и volumes** — автоматическая настройка

## 🛠 Установка

### Через пакетный менеджер:
```bash
# Ubuntu/Debian:
sudo apt update
sudo apt install docker-compose-plugin    # V2 (рекомендуется)

# Или старая версия (отдельный бинарник):
sudo apt install docker-compose           # V1

# RHEL/CentOS/Fedora:
sudo dnf install docker-compose-plugin

📄 Структура docker-compose.yml

Минимальный пример:

version: "3.8"

services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
  
  db:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: secret

Основные секции:

| Секция | Описание |
| :--- | :--- |
| `version` | Версия формата (опционально для V2) |
| `services` | Список сервисов (контейнеров) |
| `networks` | Пользовательские сети (опционально) |
| `volumes` | Именованные тома (опционально) |
| `configs` / `secrets` | Конфигурации и секреты (для Swarm) |

#⚙️ Конфигурация сервиса
Базовые параметры:

services:
  myapp:
    image: myapp:latest          # Образ (обязательно или build)
    container_name: myapp_prod   # Имя контейнера (опционально)
    
    # Сборка из Dockerfile:
    build:
      context: ./app
      dockerfile: Dockerfile.prod
      args:
        NODE_ENV: production
    
    # Порты:
    ports:
      - "80:80"                  # host:container
      - "443:443"
      - "127.0.0.1:3000:3000"    # Только localhost
    
    # Переменные окружения:
    environment:
      - DB_HOST=db
      - DB_PORT=5432
      # Или из файла:
    env_file:
      - .env
    
    # Тома:
    volumes:
      - ./data:/app/data         # bind mount
      - db_data:/var/lib/postgresql  # named volume
    
    # Сети:
    networks:
      - frontend
      - backend
    
    # Зависимости:
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    
    # Ресурсы:
    deploy:
      resources:
        limits:
          cpus: '1'
          memory: 512M
        reservations:
          memory: 256M
    
    # Restart policy:
    restart: unless-stopped
    
    # Healthcheck:
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s


#🌐 Сети и Volumes
Объявление сетей:

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
    internal: true  # Нет доступа извне

Объявление volumes:

volumes:
  db_data:
    driver: local
  cache_data:
    driver: local

Использование в сервисах:

services:
  web:
    networks:
      - frontend
      - backend
    volumes:
      - ./static:/usr/share/nginx/html:ro
  
  db:
    networks:
      - backend
    volumes:
      - db_data:/var/lib/postgresql/data

#🚀 Управление приложением
Базовые команды:

# Запустить всё в фоне:
docker compose up -d

# Запустить с пересборкой образов:
docker compose up -d --build

# Остановить (не удаляя volumes):
docker compose down

# Остановить и удалить volumes:
docker compose down -v

# Остановить и удалить всё (включая образы):
docker compose down -v --rmi all

# Просмотр логов:
docker compose logs
docker compose logs -f web      # Следить за сервисом
docker compose logs --tail 50   # Последние 50 строк

# Список запущенных сервисов:
docker compose ps

# Выполнить команду в сервисе:
docker compose exec web bash
docker compose exec db psql -U postgres

# Перезапустить сервис:
docker compose restart web

# Остановить/запустить отдельный сервис:
docker compose stop web
docker compose start web

Работа с несколькими файлами:

# Использовать несколько compose-файлов:
docker compose -f docker-compose.yml -f docker-compose.override.yml up -d

# Указать кастомный файл:
docker compose -f production.yml up -d

Переменные окружения в Compose:

# docker-compose.yml:
services:
  web:
    image: myapp:${APP_VERSION:-latest}
    ports:
      - "${WEB_PORT:-80}:80"

# .env файл (автоматически подгружается):
APP_VERSION=1.2.3
WEB_PORT=8080

# Или передать при запуске:
APP_VERSION=1.2.3 docker compose up -d

#🔄 Зависимости и порядок запуска

Базовый depends_on:

services:
  web:
    image: myapp
    depends_on:
      - db
      - redis
  
  db:
    image: postgres:15
  
  redis:
    image: redis:7

С healthcheck (рекомендуется):

services:
  db:
    image: postgres:15
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
  
  web:
    image: myapp
    depends_on:
      db:
        condition: service_healthy  # Ждём успешный healthcheck

С wait-for-it скриптом (альтернатива):

services:
  web:
    image: myapp
    command: >
      bash -c "
        ./wait-for-it.sh db:5432 --timeout=30 --strict -- 
        python app.py
      "
    depends_on:
      - db

#📦 Примеры реальных конфигураций

Пример 1: Веб-приложение + БД + Redis

version: "3.8"

services:
  postgres:
    image: postgres:15-alpine
    container_name: app_db
    environment:
      POSTGRES_USER: app
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_DB: appdb
    volumes:
      - pg_data:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    networks:
      - backend
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U app"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    container_name: app_cache
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    networks:
      - backend
    restart: unless-stopped

  web:
    build: ./app
    container_name: app_web
    ports:
      - "${WEB_PORT:-8000}:8000"
    environment:
      - DATABASE_URL=postgresql://app:${DB_PASSWORD}@postgres:5432/appdb
      - REDIS_URL=redis://redis:6379/0
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_started
    networks:
      - frontend
      - backend
    restart: unless-stopped

networks:
  frontend:
  backend:
    internal: true

volumes:
  pg_data:
  redis_data:

🧹 Очистка и обслуживание

# Удалить остановленные контейнеры:
docker compose down

# Удалить тома:
docker compose down -v

# Удалить образы, созданные при сборке:
docker compose down --rmi local

# Полная очистка:
docker compose down -v --rmi all --remove-orphans

# Просмотр использования ресурсов:
docker compose stats

# Обновить образы и пересобрать:
docker compose pull
docker compose up -d --build

# Проверить конфигурацию на ошибки:
docker compose config
docker compose config --services   # Список сервисов

#📋 Шпаргалка команд

| Задача | Команда |
| :--- | :--- |
| Запустить всё | `docker compose up -d` |
| Запустить с пересборкой | `docker compose up -d --build` |
| Остановить | `docker compose down` |
| Остановить + удалить volumes | `docker compose down -v` |
| Логи | `docker compose logs -f [service]` |
| Список сервисов | `docker compose ps` |
| Bash в сервисе | `docker compose exec service bash` |
| Перезапустить сервис | `docker compose restart service` |
| Проверить конфиг | `docker compose config` |
| Обновить образы | `docker compose pull && docker compose up -d` |
| Полная очистка | `docker compose down -v --rmi all` |

⚠️ Важные нюансы

V1 vs V2 — V2 использует docker compose (без дефиса), V1 — docker-compose.

Зависимости — depends_on не ждёт готовности сервиса, используй healthcheck.

Переменные — .env файл подгружается автоматически, но не коммить секреты в репо!

Сети — сервисы видят друг друга по имени, внешние порты пробрасывай явно.

Volumes — именованные тома сохраняются после down, удаляй с -v.

Build context — ./app копируется в контекст сборки, используй .dockerignore.
