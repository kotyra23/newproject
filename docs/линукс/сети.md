# 📌 Сети Linux: Интерфейсы и протоколы

## 🎯 Назначение

Документ описывает теоретические основы сетевых взаимодействий в Linux: модели OSI и TCP/IP, сетевые уровни, протоколы, интерфейсы и инкапсуляцию данных.

---

## 📖 Ключевые концепции

### Заголовок и протокол

- **Заголовок** — среда виртуального общения между уровнями
- **Протокол** реализует правила заголовков сообщений

### Эталонная модель сети

Эталонная модель описывает:
- Сервисы и уровни сети
- 7 уровней организации сети
- Назначение каждого уровня

**Популярные модели:**
1. **ISO OSI** (1983) — Open Systems Interconnection Reference Model
2. **TCP/IP** — фактический стандарт интернета

### 7 уровней модели OSI

| Уровень (RU) | Уровень (EN) | Единица данных |
|--------------|--------------|----------------|
| Прикладной | Application | Данные |
| Представления | Presentation | Данные |
| Сеансовый | Session | Данные |
| Транспортный | Transport | Сегмент |
| Сетевой | Network | Пакет |
| Канальный | Data Link | Кадр |
| Физический | Physical | Бит |

#### Детализация уровней:

- **Физический**: передача битов по физическим каналам
- **Канальный**: надежность передачи, доступ к среде (MAC-адреса)
- **Сетевой**: адресация (IP), маршрутизация, объединение сетей

## 🛠 Интерфейс и протокол

| Понятие | Определение |
|---------|-------------|
| **Сервис** | Что делает уровень |
| **Интерфейс** | Как использовать возможности уровня |
| **Протокол** | Как уровень реализует возможности |

**Ключевое различие:**
- **Интерфейс** — реальное взаимодействие внутри устройства
- **Протокол** — виртуальное взаимодействие между уровнями разных устройств

> 💡 Реализацию можно менять без ущерба для пользователей (принцип ООП: интерфейс публичный, реализация скрыта).

## 🔢 Адресация

### MAC-адрес

- **Размер**: 6 байт (48 бит)
- **Формат**: `1C:25:08:D2:49:45` или `1C-25-08-D2-49-45`
- **Уникальность**: гарантируется стандартом IEEE

### IP-адрес

- **IPv4**: `192.168.1.1` (4 байта)
- **IPv6**: `2001:0db8::1` (16 байт)

### Порты

| Тип | Диапазон | Примеры |
|-----|----------|---------|
| Well-known | 0–1023 | 80 (HTTP), 443 (HTTPS), 22 (SSH) |
| Registered | 1024–49151 | 3306 (MySQL), 8080 (proxy) |
| Dynamic | 49152–65535 | Эфемерные порты клиентов |

**Формат сокета**: `192.168.1.100:80`

## 📮 Транспортные протоколы

### TCP (Transmission Control Protocol)

| Характеристика | Описание |
|----------------|----------|
| Соединение | Устанавливается (3-way handshake) |
| Надежность | Гарантированная доставка |
| Порядок | Упорядоченная передача |
| Управление потоком | Есть (sliding window) |
| Применение | HTTP, SSH, SMTP, FTP |

### UDP (User Datagram Protocol)

| Характеристика | Описание |
|----------------|----------|
| Соединение | Без соединения (connectionless) |
| Надежность | Без гарантий доставки |
| Порядок | Не гарантируется |
| Накладные расходы | Минимальные |
| Применение | DNS, DHCP, VoIP, Streaming |

## 🔌 TCP: Детали

### 3-way handshake (установка соединения)

Клиент → Сервер: SYN, seq=7537

Сервер → Клиент: SYN-ACK, seq=36829, ack=7538

Клиент → Сервер: ACK, seq=7538, ack=36830

→ Соединение ESTABLISHED


### Разрыв соединения

**Плавное закрытие:**

Сторона А → Сторона Б: FIN

Сторона Б → Сторона А: ACK

Сторона Б → Сторона А: FIN

Сторона А → Сторона Б: ACK



**Принудительный разрыв:**


RST → немедленное закрытие



### Состояния TCP

| Состояние | Описание |
|-----------|----------|
| LISTEN | Ожидает входящего соединения |
| SYN_SENT | Отправлен SYN, ждем SYN-ACK |
| ESTABLISHED | Соединение активно |
| FIN_WAIT | Ожидание завершения |
| TIME_WAIT | Ожидание закрытия (2×MSL) |
| CLOSE_WAIT | Другая сторона закрыла, ждем локального close |

## 🔄 Управление потоком и перегрузкой

### Flow Control (управление потоком)

- Предотвращает "затопление" медленного получателя
- Использует **окно приема** (receive window)

### Congestion Control (управление перегрузкой)

**Алгоритм AIMD** (Additive Increase Multiplicative Decrease):


if (нет потерь):

window += 1 MSS # медленный рост

else:

window *= 0.5 # быстрое снижение

## 🌐 DNS (Domain Name System)

### Типы записей

| Тип | Назначение | Пример |
|-----|------------|--------|
| A | IPv4 адрес | `example.com → 93.184.216.34` |
| AAAA | IPv6 адрес | `→ 2606:2800:220::1` |
| CNAME | Псевдоним | `www → example.com` |
| MX | Почтовый сервер | `10 mail.example.com` |
| NS | DNS-сервер зоны | `ns1.example.com` |
| TXT | Текстовая запись | SPF, DKIM |
| PTR | Обратный поиск | `34.216.184.93.in-addr.arpa` |

### Утилиты

```bash
# Базовые запросы
nslookup example.com
dig example.com
host example.com

# Конкретный тип записи
dig example.com MX
dig -t AAAA example.com

# Обратный поиск
dig -x 93.184.216.34

# Трассировка разрешения
dig +trace example.com

# Указать сервер
dig @8.8.8.8 example.com
