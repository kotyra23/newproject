# 💾 Ошибки дисков: Диагностика

## 📊 Быстрая проверка

```bash
# Общая информация о дисках:
lsblk
fdisk -l

# SMART-статус (требует smartmontools):
sudo smartctl -H /dev/sda
sudo smartctl -a /dev/sda

# Ошибки в системном журнале:
dmesg | grep -iE "error|fail|sd[a-z]"
journalctl -k | grep -iE "I/O|sector|ext4|xfs"

#🔍 Детальная диагностика
1. Проверка SMART

# Краткий тест здоровья:
sudo smartctl -H /dev/sda

# Полная информация:
sudo smartctl -a /dev/sda

# Запустить короткий тест:
sudo smartctl -t short /dev/sda

# Запустить расширенный тест (долго!):
sudo smartctl -t long /dev/sda

# Проверить результаты теста:
sudo smartctl -l selftest /dev/sda


Интерпретация результатов:

=== START OF READ SMART DATA SECTION ===
SMART overall-health self-assessment test result: PASSED
# ✅ Диск в порядке

SMART overall-health self-assessment test result: FAILED!
# ❌ Тревога! Диск может выйти из строя

2. Проверка файловой системы

# Проверка без исправления (безопасно):
sudo fsck -n /dev/sda1

# Проверка с исправлением (требует unmount!):
sudo umount /dev/sda1
sudo fsck -y /dev/sda1

# Для XFS:
sudo xfs_repair -n /dev/sda1   # Только проверка
sudo xfs_repair /dev/sda1      # С исправлением

3. Поиск bad blocks

# Проверка на плохие блоки (недеструктивно):
sudo badblocks -sv /dev/sda1

# Запись результатов в файл:
sudo badblocks -sv -o badblocks.txt /dev/sda1

# Проверка с чтением-записью (УДАЛЯЕТ ДАННЫЕ!):
sudo badblocks -wsv /dev/sda1

4. Мониторинг ошибок в реальном времени

# Логи ядра в реальном времени:
dmesg -w

# Журнал с фильтрацией по диску:
journalctl -f | grep -i sda

# Статистика ошибок диска:
sudo iostat -x 1 | grep -E "Device|sda"

Типичные ошибки в логах:

| Сообщение | Возможная причина |
| :--- | :--- |
| `I/O error` | Физическая проблема диска/кабеля |
| `sector 12345` | Bad block на указанном секторе |
| `EXT4-fs error` | Повреждение файловой системы |
| `Buffer I/O error` | Проблема с кэшем или диском |
| `UDMA CRC error` | Неисправный SATA-кабель |

5. Проверка RAID (если используется)

# Статус RAID-массива:
cat /proc/mdstat
sudo mdadm --detail /dev/md0

# Проверка на деградацию:
sudo mdadm --detail /dev/md0 | grep -E "State|Failed"

# Просмотр событий диска в RAID:
sudo mdadm --examine /dev/sd[ab]1

#🛠 Практические сценарии
Сценарий 1: "Система тормозит, подозрение на диск"

# 1. Проверить нагрузку на диск:
iostat -x 1 5

# 2. Посмотреть ошибки в логах:
dmesg | tail -50 | grep -iE "error|sd"

# 3. Проверить SMART:
sudo smartctl -H /dev/sda

# 4. Если %util ~100% и await высокий:
#    → диск не справляется, возможно, умирает

# 5. Проверить файловую систему (после unmount):
sudo umount /mnt/data
sudo fsck -n /dev/sdb1

Сценарий 2: "Файлы не сохраняются, ошибки записи"

# 1. Проверить место и inodes:
df -h
df -i

# 2. Проверить права:
ls -ld /path/to/dir

# 3. Проверить логи на ошибки ФС:
journalctl -xb | grep -iE "ext4|xfs|error"

# 4. Проверить диск на bad blocks:
sudo badblocks -sv /dev/sdX1

# 5. Если найдены ошибки — запустить fsck:
sudo umount /dev/sdX1
sudo fsck -y /dev/sdX1

##📋 Шпаргалка команд

| Задача | Команда |
| :--- | :--- |
| Проверка дисков | `lsblk`, `fdisk -l` |
| SMART-здоровье | `smartctl -H /dev/sdX` |
| Полный SMART-отчёт | `smartctl -a /dev/sdX` |
| Запуск теста SMART | `smartctl -t short/long /dev/sdX` |
| Проверка ФС | `fsck -n /dev/sdX1` |
| Исправление ФС | `fsck -y /dev/sdX1` (после `umount`!) |
| Поиск bad blocks | `badblocks -sv /dev/sdX1` |
| Логи ядра | `dmesg \| grep -i error` |
| Журнал ошибок | `journalctl -k \| grep -i sd` |
| Статистика I/O | `iostat -x 1` |
| RAID статус | `cat /proc/mdstat`, `mdadm --detail` |
| Бэкап данных | `rsync -av /src /dst` |
