# 💾 Файловые системы

## 📀 ТИПЫ ФАЙЛОВЫХ СИСТЕМ

### Популярные ФС:

| ФС | Описание | Применение |
|----|----------|------------|
| **ext4** | Стандарт для Linux, надежная | Домашние ПК, серверы |
| **XFS** | Для больших файлов, быстрая | RHEL/CentOS, базы данных |
| **Btrfs** | Современная, снимки, сжатие | Экспериментальные задачи |
| **NTFS** | Windows совместимость | Дуал-бут, обмен данными |
| **FAT32** | Максимальная совместимость | Флешки, старые устройства |
| **exFAT** | Флешки >4GB | Внешние накопители |

## 💿 РАБОТА С ДИСКАМИ

### Просмотр дисков

```bash
lsblk                     # Список дисков (дерево)
lsblk -f                  # + ФС и метки
fdisk -l                  # Информация о разделах
blkid                     # UUID и тип ФС
df -h                     # Свободное место
du -sh folder/            # Размер папки


Разметка диска (fdisk)

sudo fdisk /dev/sdb       # Разметить диск

#Команды в fdisk:

| Команда | Описание |
| :--- | :--- |
| `n` | Новый раздел |
| `p` | Первичный раздел |
| `e` | Расширенный раздел |
| `+10G` | Размер 10GB |
| `t` | Изменить тип раздела |
| `L` | Список типов |
| `w` | Записать и выйти |
| `q` | Выйти без сохранения |
| `d` | Удалить раздел |
| `p` | Показать таблицу разделов |

## 📁 ФОРМАТИРОВАНИЕ


# ext4 (стандарт):
sudo mkfs.ext4 /dev/sdb1

# XFS (большие файлы):
sudo mkfs.xfs /dev/sdb1

# FAT32 (флешки):
sudo mkfs.vfat /dev/sdb1

# Проверка ext4:
sudo fsck.ext4 /dev/sdb1

# Проверка XFS:
sudo xfs_repair /dev/sdb1

## 📌 МОНТИРОВАНИЕ

#Временное монтирование

# Создать точку монтирования:
sudo mkdir /mnt/data

# Смонтировать раздел:
sudo mount /dev/sdb1 /mnt/data

# Проверить:
df -h | grep sdb1
mount | grep sdb1


# Опции монтирования

# Только чтение:
sudo mount -o ro /dev/sdb1 /mnt/data

# Noexec (запрет запуска):
sudo mount -o noexec /dev/sdb1 /mnt/data

# Все опции:
sudo mount -o rw,noatime,nodiratime /dev/sdb1 /mnt/data

# Размонтирование

# Стандартное:
sudo umount /mnt/data

# Если занято (найти процесс):
sudo lsof /mnt/data
sudo fuser -vm /mnt/data
sudo umount -l /mnt/data   # Ленивое размонтирование


## 🔄 АВТОМОНТИРОВАНИЕ (/etc/fstab)

# Формат строки fstab

<device>  <mount_point>  <type>  <options>  <dump>  <pass>

/dev/sdb1  /mnt/data  ext4  defaults  0  2


| Поле | Описание | Пример |
| :--- | :--- | :--- |
| `device` | Устройство или UUID | `/dev/sdb1` или `UUID=...` |
| `mount_point` | Точка монтирования | `/mnt/data` |
| `type` | Тип ФС | `ext4`, `xfs`, `vfat` |
| `options` | Опции | `defaults`, `noatime`, `ro` |
| `dump` | Резервное копирование | `0` (нет), `1` (есть) |
| `pass` | Порядок fsck | `0` (нет), `1` (root), `2` (остальные) |

#Как добавить запись:

# 1. Узнать UUID:
sudo blkid /dev/sdb1
# UUID="abc123..." TYPE="ext4"

# 2. Открыть fstab:
sudo nano /etc/fstab

# 3. Добавить строку:
UUID=abc123...  /mnt/data  ext4  defaults  0  2

# 4. Проверить (ВАЖНО!):
sudo mount -a
# Если ошибок нет — всё ок!


## 🧱 LVM (Logical Volume Manager)

#Основные понятия

| Аббревиатура | Расшифровка | Описание |
| :--- | :--- | :--- |
| **PV** | Physical Volume | Физический том (диск/раздел) |
| **VG** | Volume Group | Группа томов (пул дисков) |
| **LV** | Logical Volume | Логический том (раздел) |

Создание LVM

# 1. Создать Physical Volume:
sudo pvcreate /dev/sdb1

# 2. Создать Volume Group:
sudo vgcreate myvg /dev/sdb1

# 3. Создать Logical Volume (10GB):
sudo lvcreate -L 10G -n mylv myvg

# 4. Отформатировать:
sudo mkfs.ext4 /dev/myvg/mylv

# 5. Смонтировать:
sudo mkdir /mnt/lvm
sudo mount /dev/myvg/mylv /mnt/lvm

## Изменение размера

# Увеличить LV на 5GB:
sudo lvextend -L +5G /dev/myvg/mylv

# Применить изменения к ФС:
sudo resize2fs /dev/myvg/mylv    # для ext4
# или
sudo xfs_growfs /mnt/lvm         # для XFS

# Уменьшить (ТОЛЬКО offline для ext4!):
sudo umount /mnt/lvm

sudo e2fsck -f /dev/myvg/mylv

sudo resize2fs /dev/myvg/mylv 5G

sudo lvreduce -L 5G /dev/myvg/mylv

sudo mount /dev/myvg/mylv /mnt/lvm

#Добавление диска в LVM

# Новый диск /dev/sdc1:
sudo pvcreate /dev/sdc1
sudo vgextend myvg /dev/sdc1

# Теперь можно увеличить LV:
sudo lvextend -L +20G /dev/myvg/mylv

sudo resize2fs /dev/myvg/mylv

## 🛠 Практические команды
Диагностика дисков

# Информация о диске:
sudo hdparm -I /dev/sda

# SMART данные:
sudo smartctl -a /dev/sda

# Скорость чтения:
sudo hdparm -t /dev/sda

#Поиск больших файлов

# Файлы >100MB:
sudo find / -type f -size +100M -exec ls -lh {} \;

# Топ папок по размеру:
sudo du -h / | grep -E '^[0-9\.]+G' | sort -rh | head -20

#Очистка места

# Очистить кэш apt:
sudo apt clean
sudo apt autoclean

# Удалить старые ядра:
sudo apt autoremove --purge

# Очистить логи:
sudo journalctl --vacuum-time=7d
