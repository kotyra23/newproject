    🐳 Docker: Основы¶

    📖 Что такое Docker?¶

    Docker — платформа для разработки, доставки и запуска приложений в контейнерах.
    Контейнер — изолированная среда, которая содержит всё необходимое для работы приложения: код, библиотеки, зависимости, настройки.

    Ключевые преимущества:¶

    • ✅ Изоляция: приложения не конфликтуют друг с другом
    • ✅ Переносимость: работает одинаково на любом хосте ("работает на моём ноутбуке → работает в продакшене")
    • ✅ Лёгкость: контейнеры делят ядро ОС, в отличие от тяжёлых ВМ
    • ✅ Быстрый запуск: секунды вместо минут
    • ✅ Версионирование: образы хранятся в реестрах, легко откатываться

    🧩 Основные понятия¶

    Термин Описание
    Image (Образ) Готовый шаблон с ОС, зависимостями и кодом (только для чтения)
    Container (Контейнер) Запущенный экземпляр образа (с изменяемым слоем)
    Dockerfile Текстовый файл с инструкциями для сборки образа
    Registry (Реестр) Хранилище образов (Docker Hub, GitLab Registry, Harbor)
    Volume (Том) Постоянное хранилище данных вне контейнера
    Network (Сеть) Виртуальная сеть для связи контейнеров

    🛠 Установка (Linux)¶

    ```bash

    Официальный скрипт установки:¶

    curl -fsSL https://get.docker.com | sh

    Добавить пользователя в группу docker (чтобы не использовать sudo):¶

    sudo usermod -aG docker $USER newgrp docker

    Проверить установку:¶

    docker --version docker run hello-world

    📦 Работа с образами¶

    Поиск в Docker Hub:¶

    docker search nginx

    Скачать образ:¶

    docker pull nginx:latest docker pull ubuntu:22.04

    Список локальных образов:¶

    docker images docker image ls

    Удалить образ:¶

    docker rmi nginx:latest docker image prune -a # Удалить все неиспользуемые образы

    История слоёв:¶

    docker history nginx:latest

    Сохранить/загрузить образ в файл:¶

    docker save -o nginx.tar nginx:latest docker load -i nginx.tar

    Тегирование и отправка в реестр:

    docker tag nginx:latest myregistry.com/nginx:v1.0

    docker push myregistry.com/nginx:v1.0

    🏃 Запуск контейнеров¶

    Базовый запуск:

    docker run nginx Полезные флаги docker run:

    Флаг Описание
    -d Запуск в фоне (detached)
    -it Интерактивный режим + TTY (для bash)
    --name Задать имя контейнера
    --rm Автоматически удалить после остановки
    -p host:container Проброс порта
    -v host:container Монтирование тома
    --restart Политика перезапуска (no, always, on-failure, unless-stopped)
    --env / -e Переменные окружения
    --network Подключить к сети

    Примеры:

    Веб-сервер в фоне с пробросом порта:¶

    docker run -d --name web -p 8080:80 nginx

    Интерактивный bash в Ubuntu:¶

    docker run -it --rm ubuntu:22.04 bash

    Контейнер с переменной и ограничением памяти:¶

    docker run -d --name app \ -e DB_HOST=postgres \ -m 512m \ --restart unless-stopped \ myapp:latest

    🔄 Управление контейнерами¶

    Список запущенных:¶

    docker ps docker container ls

    Все контейнеры (включая остановленные):¶

    docker ps -a

    Остановить/запустить/перезапустить:¶

    docker stop web docker start web docker restart web

    Удалить контейнер:¶

    docker rm web docker rm -f web # Принудительно (если запущен)

    Логи контейнера:¶

    docker logs web docker logs -f web # Следить в реальном времени docker logs --tail 50 web # Последние 50 строк

    Выполнить команду внутри запущенного контейнера:¶

    docker exec -it web bash docker exec web nginx -t

    Копирование файлов:¶

    docker cp web:/etc/nginx/nginx.conf ./nginx.conf docker cp ./config.conf web:/etc/nginx/conf.d/

    💾 Хранение данных (Volumes & Bind Mounts)¶

    Создать:¶

    docker volume create mydata

    Список и информация:¶

    docker volume ls docker volume inspect mydata

    Использовать в контейнере:¶

    docker run -d --name db -v mydata:/var/lib/mysql mysql:8

    Bind Mounts (прямое монтирование папки хоста):

    docker run -d --name web \ -v $(pwd)/html:/usr/share/nginx/html:ro \ -p 80:80 \ nginx

    Очистка:

    docker volume prune # Удалить неиспользуемые тома

    Сети Docker¶

    Список сетей:¶

    docker network ls

    Информация о сети:¶

    docker network inspect bridge

    Создать свою сеть:¶

    docker network create mynet

    Запустить контейнер в сети:¶

    docker run -d --name app1 --network mynet myapp docker run -d --name app2 --network mynet postgres

    Подключить запущенный контейнер:¶

    docker network connect mynet web

    Проброс портов (-p):

    host:container¶

    -p 8080:80 # Локальный 8080 → контейнерный 80 -p 0.0.0.0:80:80 # На всех интерфейсах -p 127.0.0.1:80:80 # Только localhost

    📄 Dockerfile: Основы¶

    Пример минимального Dockerfile:

    1. Базовый образ¶

    FROM python:3.11-slim

    2. Рабочая директория¶

    WORKDIR /app

    3. Копирование зависимостей и установка¶

    COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt

    4. Копирование кода¶

    COPY . .

    5. Переменные окружения¶

    ENV PORT=8000

    6. Проброс порта (документация)¶

    EXPOSE 8000

    7. Команда запуска¶

    CMD ["python", "app.py"]

    Основные инструкции:

    Инструкция Описание
    FROM Базовый образ (обязательно первой)
    WORKDIR Рабочая директория внутри контейнера
    COPY Копирование файлов с хоста
    ADD Как COPY, но поддерживает URL и авто-распаковку архивов
    RUN Выполнение команды при сборке (создаёт слой)
    CMD Команда по умолчанию при запуске (переопределяется)
    ENTRYPOINT Основная команда контейнера (не переопределяется легко)
    ENV Переменные окружения
    EXPOSE Документирование портов (не пробрасывает!)
    USER Пользователь для запуска
    HEALTHCHECK Проверка здоровья контейнера

    Сборка и запуск:

    docker build -t myapp:1.0 . docker run -d --name myapp -p 8000:8000 myapp:1.0

    🧹 Очистка системы

    Удалить всё остановленное, все сети, все образы без контейнеров:¶

    docker system prune

    Полная очистка (включая тома!):¶

    docker system prune -a --volumes

    Проверить占用 место:¶

    docker system df

    ⚠️ Важные нюансы

    Dockerfile кэширует слои — меняй порядок инструкций: сначала зависимости, потом код.

    Используй .dockerignore — чтобы не копировать node_modules, .git, venv.

    Не запускай от root — добавляй USER nobody или создавай пользователя.

    EXPOSE не открывает порты — только документирует. Проброс через -p или docker-compose.

    CMD vs ENTRYPOINT — ENTRYPOINT задаёт исполняемый файл, CMD — аргументы по умолчанию.

    Ресурсные лимиты — всегда задавай -m и --cpus в проде.
