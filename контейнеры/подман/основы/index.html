    🦦 Podman: Основы¶

    📖 Что такое Podman?¶

    Podman (POD Manager) — инструмент для управления контейнерами и подами, без демона (daemonless).
    Разработан Red Hat как альтернатива Docker с акцентом на безопасность и стандарты.

    Ключевые отличия от Docker:¶

    Характеристика Docker Podman
    Архитектура Клиент-сервер (демон dockerd) Форк процессов (без демона)
    Rootless Сложная настройка Поддержка из коробки
    Systemd Через внешние скрипты Нативная генерация systemd юнитов
    Pods Только через Kubernetes / Compose Нативная поддержка (podman pod)
    Совместимость — Алиас docker=podman работает в 95% случаев

    Преимущества:¶

    • ✅ Без демона — нет единой точки отказа, проще мониторинг
    • ✅ Rootless по умолчанию — контейнеры запускаются от пользователя
    • ✅ Systemd-интеграция — podman generate systemd создаёт юниты
    • ✅ Pods — группировка контейнеров как в Kubernetes
    • ✅ Docker-compatible CLI — те же команды, тот же синтаксис

    🛠 Установка (Linux)¶

    RHEL / CentOS / Fedora:¶

    ```bash sudo dnf install podman podman-docker

    Debian / Ubuntu:

    Добавить репозиторий (если нужен свежий):¶

    echo "deb https://download.opensuse.org/repositories/devel:/kubic:/libcontainers:/stable/xUbuntu_$(lsb_release -rs)/ /" | sudo tee /etc/apt/sources.list.d/devel:kubic:libcontainers:stable.list curl -fsSL https://download.opensuse.org/repositories/devel:kubic:libcontainers:stable/xUbuntu_$(lsb_release -rs)/Release.key | gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/devel_kubic_libcontainers_stable.gpg > /dev/null

    sudo apt update sudo apt install podman podman-docker

    🔄 Совместимость с Docker¶

    Podman использует тот же CLI, что и Docker. В большинстве случаев можно просто заменить docker на podman:

    Docker:¶

    docker run -d --name web -p 80:80 nginx

    Podman (то же самое!):¶

    podman run -d --name web -p 80:80 nginx

    Настройка алиаса (опционально):

    ~/.bashrc или ~/.zshrc:¶

    alias docker=podman

    Или системно:¶

    sudo ln -s /usr/bin/podman /usr/local/bin/docker

    📦 Работа с образами¶

    Поиск (использует registry.redhat.io и Docker Hub):¶

    podman search nginx

    Скачать образ:¶

    podman pull nginx:latest podman pull quay.io/libpod/alpine:latest # Из альтернативного реестра

    Список образов:¶

    podman images podman image ls

    Удалить образ:¶

    podman rmi nginx:latest podman image prune -a

    История слоёв:¶

    podman history nginx:latest

    Сохранить/загрузить:¶

    podman save -o nginx.tar nginx:latest podman load -i nginx.tar

    Тегирование:¶

    podman tag nginx:latest myregistry.com/nginx:v1.0 podman push myregistry.com/nginx:v1.0

    🏃 Запуск контейнеров Базовый запуск (синтаксис как в Docker):

    podman run nginx

    Полезные флаги (идентичны Docker):

    Флаг Описание
    -d Запуск в фоне
    -it Интерактивный режим + TTY
    --name Имя контейнера
    --rm Удалить после остановки
    -p host:container Проброс порта
    -v host:container Монтирование тома
    --restart Политика перезапуска
    -e Переменные окружения
    --network Сеть

    Примеры:

    Веб-сервер:¶

    podman run -d --name web -p 8080:80 nginx

    Интерактивный shell:¶

    podman run -it --rm alpine sh

    Rootless с привязкой к порту >1024:¶

    podman run -d --name app -p 8080:8080 myapp:latest

    С ограничением ресурсов:¶

    podman run -d --name limited \ --memory 512m \ --cpus 1.5 \ myapp:latest

    🔄 Управление контейнерами¶

    Список запущенных:¶

    podman ps podman container ls

    Все контейнеры:¶

    podman ps -a

    Остановить/запустить:¶

    podman stop web podman start web podman restart web

    Удалить:¶

    podman rm web podman rm -f web # Принудительно

    Логи:¶

    podman logs web podman logs -f web podman logs --tail 100 web

    Выполнить команду внутри:¶

    podman exec -it web bash podman exec web nginx -t

    Копирование файлов:¶

    podman cp web:/etc/nginx/nginx.conf ./nginx.conf podman cp ./config.conf web:/etc/nginx/

    Rootless нюансы:

    Проброс портов <1024 требует прав или настройки:¶

    Вариант 1: использовать порты >=1024¶

    podman run -p 8080:80 nginx

    Вариант 2: настроить sysctl (требует root один раз):¶

    sudo sysctl net.ipv4.ip_unprivileged_port_start=0

    Проверить, под каким пользователем запущен контейнер:¶

    podman inspect web --format '{{.User}}'

    💾 Хранение данных (Volumes)¶

    Создать том:¶

    podman volume create mydata

    Список томов:¶

    podman volume ls

    Информация:¶

    podman volume inspect mydata

    Использовать:¶

    podman run -d --name db -v mydata:/var/lib/mysql postgres:15

    Bind mount (как в Docker):¶

    podman run -d --name web \ -v $(pwd)/html:/usr/share/nginx/html:ro \ -p 80:80 \ nginx

    Очистка:¶

    podman volume prune

    🌐 Сети¶

    Список сетей:¶

    podman network ls

    Информация:¶

    podman network inspect bridge

    Создать сеть:¶

    podman network create mynet

    Запустить в сети:¶

    podman run -d --name app1 --network mynet myapp podman run -d --name app2 --network mynet postgres

    Подключить запущенный:¶

    podman network connect mynet web

    🦫 Pods (группы контейнеров)¶

    Pod — группа контейнеров, разделяющих сеть и volumes (как в Kubernetes).

    Создать pod:¶

    podman pod create --name mypod -p 8080:80

    Запустить контейнер в pod:¶

    podman run -d --pod mypod --name web nginx podman run -d --pod mypod --name app myapp:latest

    Управление pod:¶

    podman pod ps podman pod stop mypod podman pod start mypod podman pod rm mypod

    Удалить pod со всеми контейнерами:¶

    podman pod rm -f mypod

    Пример: веб-приложение + БД в одном pod:

    Создать pod с пробросом порта:¶

    podman pod create --name webapp -p 3000:3000

    Запустить БД:¶

    podman run -d --pod webapp --name postgres \ -e POSTGRES_PASSWORD=secret \ postgres:15

    Запустить приложение (видит БД по localhost):¶

    podman run -d --pod webapp --name backend \ -e DB_HOST=localhost \ myapp:latest

    📄 Podman и Dockerfile¶

    Podman полностью совместим с Dockerfile. Сборка:

    Стандартная сборка:¶

    podman build -t myapp:1.0 .

    С использованием buildah (бэкенд по умолчанию):¶

    podman build --format docker -t myapp:1.0 .

    Без кэша:¶

    podman build --no-cache -t myapp:1.0 .

    С дополнительными реестрами:¶

    podman build --additional-tag myregistry.com/myapp:latest -t myapp:1.0 .

    FROM python:3.11-slim WORKDIR /app COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt COPY . . ENV PORT=8000 EXPOSE 8000 USER nobody CMD ["python", "app.py"]

    ⚙️ Интеграция с systemd

    Podman может генерировать systemd-юниты для автозапуска контейнеров:

    Создать юнит для контейнера:¶

    podman generate systemd --name web --files --new

    Результат: файл ~/container-web.service¶

    Установить в systemd:¶

    mkdir -p ~/.config/systemd/user/ cp ~/container-web.service ~/.config/systemd/user/ systemctl --user daemon-reload systemctl --user enable --now container-web.service

    Проверить статус:¶

    systemctl --user status container-web.service

    Флаги generate systemd:

    Флаг Описание
    --name Имя контейнера или pod
    --files Создать файлы .service, а не выводить в stdout
    --new Использовать podman run в ExecStart (а не start по имени)
    --restart-policy Задать политику (no, on-failure, always)

    🧹 Очистка системы

    Удалить остановленные контейнеры:¶

    podman container prune

    Удалить неиспользуемые образы:¶

    podman image prune

    Удалить неиспользуемые тома:¶

    podman volume prune

    Полная очистка:¶

    podman system prune

    Полная очистка с томами:¶

    podman system prune -a --volumes

    Проверить использование места:¶

    podman system df

    ⚠️ Важные нюансы

    Rootless по умолчанию — порты <1024 требуют настройки ip_unprivileged_port_start.

    Нет демона — podman ps показывает только контейнеры текущего пользователя.

    Storage driver — по умолчанию overlay, как в Docker.

    Network driver — rootless использует slirp4netns (может быть медленнее).

    Podman Compose — для docker-compose.yml используй podman-compose (отдельный пакет).

    SELinux — в RHEL/Fedora может требовать :Z или :z для bind mounts.
