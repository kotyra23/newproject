    🌐 Сетевые задержки: Диагностика¶

    📊 Быстрая проверка¶

    ```bash

    Проверка доступности и пинга:¶

    ping -c 4 8.8.8.8 ping -c 4 example.com

    Трассировка маршрута:¶

    traceroute 8.8.8.8 tracepath example.com

    Проверка разрешения имён:¶

    nslookup example.com dig example.com +short

    🔍 Детальная диагностика¶

    1. Проверка локальной сети

    Статус интерфейса:¶

    ip link show eth0 ethtool eth0

    Статистика интерфейса:¶

    ip -s link show eth0 cat /proc/net/dev

    Ошибки и дропы:¶

    ifconfig eth0 | grep -E "errors|dropped|overruns"

    Признаки проблем:

    RX/TX errors > 0 → проблемы с кабелем, портом, драйвером

    dropped → переполнение буфера, нехватка ресурсов

    collisions (устар.) → проблемы в half-duplex сети

    1. Проверка маршрутизации

    Таблица маршрутизации:¶

    ip route show route -n

    Проверка шлюза:¶

    ip route get 8.8.8.8

    Трассировка с деталями:¶

    mtr -r -c 10 8.8.8.8

    1. Проверка соединений и портов

    Активные соединения:¶

    ss -tulnp ss -tn state established | wc -l

    Проверка конкретного порта:¶

    nc -zv host port telnet host port

    Статистика TCP:¶

    netstat -st ss -s

    1. Проверка DNS

    Время разрешения имени:¶

    time nslookup example.com

    Проверка разных серверов:¶

    dig @8.8.8.8 example.com dig @1.1.1.1 example.com

    Проверка кэша (если используется nscd/systemd-resolved):¶

    systemd-resolve --statistics

    1. Проверка пропускной способности

    Тест скорости (требует установки):¶

    iperf3 -c server_ip speedtest-cli

    Локальная пропускная способность:¶

    iftop -i eth0 nload eth0

    🛠 Практические сценарии¶

    Сценарий 1: "Сайт медленно грузится"

    1. Проверить пинг до сервера:¶

    ping -c 10 example.com

    2. Проверить маршрутизацию:¶

    mtr -r -c 20 example.com

    3. Проверить DNS:¶

    time dig example.com +short

    4. Проверить соединение на порт 80/443:¶

    time curl -o /dev/null -s -w "%{time_total}\n" https://example.com

    5. Проверить локальную сеть:¶

    ip -s link show eth0 | grep -E "RX|TX"

    Сценарий 2: "SSH подключается с задержкой"

    1. Проверить пинг до хоста:¶

    ping -c 5 host

    2. Проверить разрешение имени (если используешь имя):¶

    time ssh user@host echo test

    3. Отключить DNS-проверки в SSH (клиент):¶

    В ~/.ssh/config добавить:¶

    UseDNS no¶

    4. Проверить аутентификацию:¶

    ssh -vvv user@host 2>&1 | grep -E "debug1:|Authentications"

    5. Проверить GSSAPI (частая причина задержек):¶

    В /etc/ssh/ssh_config добавить:¶

    GSSAPIAuthentication no¶

    Сценарий 3: "Потери пакетов в локальной сети"

    1. Проверить интерфейс на ошибки:¶

    ethtool -S eth0 | grep -E "err|drop"

    2. Проверить кабель/порт:¶

    ethtool eth0 | grep -E "Speed|Duplex|Link"

    3. Проверить на коллизии (старые сети):¶

    ifconfig eth0 | grep collisions

    4. Тест с большим пакетом:¶

    ping -s 1472 -c 10 gateway_ip # MTU тест

    5. Проверить switch/роутер:¶

    (требует доступа к оборудованию)¶

    📋 Шпаргалка команд¶

    Задача Команда
    Проверка пинга ping -c 10 host
    Трассировка traceroute host, mtr host
    Проверка DNS dig host, nslookup host
    Статус интерфейса ip link show, ethtool eth0
    Статистика интерфейса ip -s link, cat /proc/net/dev
    Активные соединения ss -tulnp, netstat -tulpn
    Проверка порта nc -zv host port, telnet host port
    TCP-статистика netstat -st, ss -s
    Пропускная способность iperf3, iftop, nload
    SSH с отладкой ssh -vvv user@host

    ⚠️ Частые ошибки

    Не путай потерю пакетов и высокую задержку — это разные проблемы.

    Проверяй локально сначала — проблема может быть в твоём сегменте сети.

    DNS может быть узким местом — тестируй с разными серверами.

    MTU проблемы — пинг работает, а HTTP нет? Проверь фрагментацию.

    Firewall может дропать пакеты — проверяй iptables -L -v -n.
